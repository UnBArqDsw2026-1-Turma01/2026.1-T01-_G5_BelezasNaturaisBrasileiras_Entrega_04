# Etapa 1 - Entender

A etapa **Entender** e o ponto de partida do Design Sprint.
Nela, o time busca compreender o problema de forma ampla antes de propor solucoes.

## Objetivo

Construir uma visao compartilhada sobre:
- quem sao os usuarios
- qual dor principal precisa ser resolvida
- quais restricoes de negocio e tecnologia existem
- quais resultados se espera atingir

## O que normalmente acontece nesta etapa

- alinhamento entre as pessoas envolvidas no projeto
- levantamento de dados e contexto
- mapeamento da jornada ou fluxo principal
- identificacao de riscos, duvidas e oportunidades

## Resultado esperado

Ao final da etapa Entender, o grupo deve ter clareza sobre o desafio certo a atacar no Sprint,
com foco definido e criterios para guiar as proximas etapas.
