# Etapa 1 - Entender

A etapa Entender é o ponto de partida do Design Sprint. Nela, o time busca compreender o problema de forma ampla antes de propor soluções.

## Objetivo
Construir uma visão compartilhada sobre:
*   quem são os usuários
*   qual dor principal precisa ser resolvida
*   quais restrições de negócio e tecnologia existem
*   quais resultados se espera atingir

## O que acontece nesta etapa
*   alinhamento entre as pessoas envolvidas no projeto
*   levantamento de dados e contexto
*   mapeamento da jornada ou fluxo principal
*   identificação de riscos, dúvidas e oportunidades

## Resultado
Compreendemos o problema central: a dificuldade de encontrar trilhas e eventos turisticos organizadas e seguras em um ambiente de informações dispersas. O público-alvo foi definido como aventureiros, guias locais, organizadores de eventos naturais assim como guias turisticos e peregrinos religiosos.

Realizamos um brainstorming verbal para mapear dores, como a dificuldade de encontrar trilhas em pontos turísticos específicos e a atual desvalorização das riquezas naturais do Brasil devido à falta de uma plataforma centralizada que valide a idoneidade dos guias.

Identificamos os seguintes norteadores (How Might We):
*   Como poderíamos garantir que o usuário não realize pagamentos indevidos antes da trilha?
*   Como poderíamos manter a base de pontos turísticos confiável sem centralizar todo o trabalho nos administradores?
*   Como poderíamos validar se alguém realmente compareceu à trilha para evitar perfis com reputação falsa?
