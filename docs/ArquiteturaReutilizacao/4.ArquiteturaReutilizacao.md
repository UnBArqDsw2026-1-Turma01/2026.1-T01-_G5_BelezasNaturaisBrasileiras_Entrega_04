# 4. Arquitetura & Reutilização de Software

## Introdução

Este módulo documenta a arquitetura e as práticas de reutilização de software da plataforma **Belezas Naturais Brasileiras**. O sistema é composto por um backend NestJS 11 e um frontend React 18, organizados em monorepo.

## Foco Escolhido

A equipe optou pelo **4.1 — Documento de Arquitetura de Software (DAS)**, utilizando o modelo de visões 4+1 de Kruchten como estrutura de representação.

A escolha se justifica pelo seguinte: o projeto possui módulos verticais bem delimitados, camadas com regras de dependência explícitas e padrões GoF implementados no código — todos elementos que tornam a especificação das visões arquiteturais factível e rastreável ao código-fonte real.

## Metodologia

A documentação segue o modelo **4+1** (Kruchten, 1995), que organiza a arquitetura em cinco perspectivas complementares:

| Visão         | Foco                                                            | Audiência principal       |
| ------------- | --------------------------------------------------------------- | ------------------------- |
| Lógica        | Estrutura de pacotes, classes e padrões                         | Desenvolvedores           |
| Casos de Uso  | Cenários que direcionaram as decisões arquiteturais             | Todos os stakeholders     |
| Processo      | Fluxos de execução, concorrência e ciclo de vida de requisições | Desenvolvedores, operação |
| Implementação | Organização do código-fonte, componentes e estratégia de build  | Desenvolvedores           |
| Implantação   | Topologia de nós, serviços e protocolos de comunicação          | Operação, DevOps          |
| Dados         | Modelo relacional e evolução do esquema                         | Desenvolvedores, DBA      |

> Nota: o modelo original 4+1 não inclui a Visão de Dados como visão canônica. A equipe adicionou essa visão por ser relevante para o projeto — o esquema relacional é a fonte de dados de todas as entidades do domínio.

## Senso Crítico

A entrega permitia escolher entre DAS (4.1) e Reutilização de Software (4.2). A equipe escolheu o DAS porque o material disponível favorece esse foco: a arquitetura em camadas e os padrões implementados nas fases anteriores fornecem conteúdo verificável para as seis visões. A reutilização não ficou de fora — o documento [4.2](4.2.ReutilizacaoDeSoftware.md) registra as práticas de reutilização do projeto como material complementar, sem concorrer com o foco principal.

O trade-off da decisão: especificar seis visões (em vez das duas mínimas) aumenta o volume de documentação a manter. A equipe considerou o custo aceitável porque os documentos foram gerados a partir do código real, reduzindo o esforço de produção, e porque a cobertura adicional fortalece a rastreabilidade exigida na avaliação.

## Declaração de Uso de IA

Este documento foi desenvolvido com o auxílio de IA para otimizar a estrutura e a apresentação do conteúdo. Todas as decisões de arquitetura, escolhas de foco e rastreabilidade com artefatos anteriores foram realizadas pela equipe com senso crítico e autoridade própria.

A IA foi utilizada como ferramenta de suporte na documentação: estruturação do módulo, organização das seções e geração de descrições baseadas no código implementado e nos artefatos das fases anteriores.

Cada seção foi revisada e ajustada conforme as necessidades do projeto. A equipe mantém total responsabilidade pelas escolhas implementadas.
