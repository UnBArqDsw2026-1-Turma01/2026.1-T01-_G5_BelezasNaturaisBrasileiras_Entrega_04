## Orientações para Preenchimento

Para que a contribuição seja aceita como comprobatório:

- **Commit/PR:** link direto para o commit ou pull request com a contribuição
- **Revisão documentada:** link para o PR onde o membro aparece como revisor com comentário
- **Reunião:** print da reunião com data e pauta, adicionado em `docs/assets/comprobatorios/`
- **Debate registrado:** issue ou comentário de PR com ponto de vista sobre uma decisão arquitetural

Significâncias possíveis: `Excelente` · `Boa` · `Regular` · `Ruim` · `Nula`
