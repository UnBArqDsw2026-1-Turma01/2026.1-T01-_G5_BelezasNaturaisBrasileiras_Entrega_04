# 3.2 Padrões de Projeto Estruturais (GoFs Estruturais)

## Introdução

Os Padrões de Projeto Estruturais (Structural Design Patterns) são soluções que lidam com a composição de classes e objetos para formar estruturas maiores, mantendo essas estruturas flexíveis e eficientes. Enquanto os padrões criacionais focam em como criar objetos, os padrões estruturais focam em como compor objetos e classes em estruturas maiores, reduzindo dependências e facilitando a manutenção.

Os padrões estruturais são essenciais para criar sistemas onde múltiplos componentes precisam trabalhar em conjunto de forma harmoniosa. Eles permitem que você combine objetos de diferentes formas sem acoplar o código cliente a implementações específicas, facilitando a extensão e a modificação do sistema sem breaking changes.


## Padrões Estruturais

| Padrão | Descrição | Onde Aplicar no Código | Benefícios |
| :--- | :--- | :--- | :--- |
| **Adapter** | Converte a interface de uma classe em outra esperada pelos clientes, permitindo que classes incompatíveis trabalhem juntas. | | |
| **Bridge** | Desacopla uma abstração de sua implementação, permitindo que ambas variem independentemente. | | |
| **Composite** | Compõe objetos em estruturas em árvore para representar hierarquias parte-todo, deixando os clientes tratarem objetos individuais e composições de forma uniforme. | | |
| **Decorator** | Adiciona responsabilidades a um objeto dinamicamente, fornecendo uma alternativa flexível ao uso de subclasses. | | |
| **Facade** | Fornece uma interface unificada e simplificada para um conjunto de interfaces em um subsistema. | | |
| **Flyweight** | Usa compartilhamento para sustentar eficientemente grandes quantidades de objetos finos, economizando memória. | | |
| **Proxy** | Fornece um substituto ou intermediário para outro objeto, controlando o acesso a ele. | | |
