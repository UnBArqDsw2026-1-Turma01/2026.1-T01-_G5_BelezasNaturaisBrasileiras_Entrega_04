## Introdução

O **Proxy** é um padrão estrutural que fornece um substituto ou intermediário para outro objeto, controlando o acesso a ele. É útil quando você deseja adicionar funcionalidades como lazy loading, caching ou controle de acesso sem modificar o objeto original.

Este padrão cria um objeto representante que controla como o objeto real é acessado.

## Quando Aplicar?

- Quando você precisa de um substituto ou intermediário para outro objeto
- Quando lazy loading é necessário (criar objetos apenas quando necessário)
- Quando você deseja controlar e monitorar o acesso a um objeto
- Quando você precisa adicionar funcionalidades como logging, cache ou sincronização
- Quando objetos remotos precisam ser acessados localmente

## Metodologia

Durante o desenvolvimento do backend do projeto Belezas Naturais Brasileiras, identificamos que a verificação de autorização (_"apenas o organizador pode finalizar a trilha"_) estava acoplada ao `FinalizarTrilhaUseCase`, misturando lógica de negócio com preocupação de segurança. Esse acoplamento viola o Princípio da Responsabilidade Única e dificulta testar cada responsabilidade de forma isolada.

A solução foi aplicar o **Protection Proxy**: um `TrilhaProxyRepository` que intercepta chamadas de escrita ao `ITrilhaRepository` e verifica a identidade do requisitante antes de delegar à cadeia real. Para propagar o userId sem alterar a assinatura dos métodos da interface (que recebem apenas `Trilha`), utilizamos `AsyncLocalStorage` — o mecanismo nativo do Node.js para contexto por requisição, thread-safe e sem dependências externas.

A composição final da cadeia de repositórios é: `TrilhaProxyRepository → AuditedTrilhaRepository → CachedTrilhaRepository → PrismaTrilhaRepository`.

A implementação seguiu a metodologia **TDD (Test-Driven Development)**, com os arquivos de teste escritos antes da implementação:

1. Escrita do spec (`TrilhaProxyRepository.spec.ts`) cobrindo os cenários de autorização — fase **RED**
2. Implementação do `TrilhaRequestContext` e `TrilhaProxyRepository` para fazer os testes passarem — fase **GREEN**
3. Remoção da verificação de autorização do `FinalizarTrilhaUseCase` e atualização do controller e do módulo — fase **REFACTOR**

## Estrutura e Participantes

| Classe                    | Papel no Padrão            | Responsabilidade                                                                                                                           |
| :------------------------ | :------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| `ITrilhaRepository`       | **Subject** (interface)    | Define o contrato `create`, `findById`, `findAll`, `save` — implementado tanto pelo Proxy quanto pelo objeto real                          |
| `TrilhaProxyRepository`   | **Proxy**                  | Implementa `ITrilhaRepository`; intercepta `save()` e verifica se o `requesterId` do contexto é o organizador da trilha antes de delegar   |
| `AuditedTrilhaRepository` | **Real Subject** (parcial) | Primeira camada da cadeia real após o Proxy — adiciona auditoria de operações                                                              |
| `TrilhaRequestContext`    | **Supporting Service**     | Armazena o `requesterId` do usuário autenticado via `AsyncLocalStorage`; é thread-safe e propagado automaticamente em chamadas assíncronas |
| `TrilhasController`       | **Client**                 | Define o contexto antes de delegar ao Facade: `context.run(userId, () => facade.finalizar(...))`                                           |
| `FinalizarTrilhaUseCase`  | **Subsystem**              | Após refatoração, foca exclusivamente na lógica de negócio (finalizar trilha e emitir evento); autorização removida para o Proxy           |

## Descrição das Classes

### `TrilhaProxyRepository`

**Caminho no Projeto:** [`backend/src/modules/trilhas/infrastructure/persistence/TrilhaProxyRepository.ts`](../../../backend/src/modules/trilhas/infrastructure/persistence/TrilhaProxyRepository.ts)

Proxy de proteção que controla o acesso de escrita ao `ITrilhaRepository`. Intercepta o método `save()` e verifica se o `requesterId` presente no `TrilhaRequestContext` corresponde ao `organizadorId` da trilha sendo modificada. Operações de leitura e criação são delegadas livremente.

#### Métodos

| Método       | Parâmetros       | Retorno                   | Comportamento                                                                                                                     |
| :----------- | :--------------- | :------------------------ | :-------------------------------------------------------------------------------------------------------------------------------- |
| `save()`     | `trilha: Trilha` | `Promise<Trilha>`         | Lê `requesterId` do contexto; lança `ForbiddenException` se ausente ou diferente de `trilha.organizadorId`; caso contrário delega |
| `create()`   | `trilha: Trilha` | `Promise<Trilha>`         | Delega diretamente ao wrapped sem verificação (qualquer usuário autenticado pode criar)                                           |
| `findById()` | `id: string`     | `Promise<Trilha \| null>` | Delega diretamente ao wrapped sem verificação                                                                                     |
| `findAll()`  | —                | `Promise<Trilha[]>`       | Delega diretamente ao wrapped sem verificação                                                                                     |

---

### `TrilhaRequestContext`

**Caminho no Projeto:** [`backend/src/modules/trilhas/domain/services/TrilhaRequestContext.ts`](../../../backend/src/modules/trilhas/domain/services/TrilhaRequestContext.ts)

Serviço injectable que usa `AsyncLocalStorage` do Node.js para propagar o `requesterId` do usuário autenticado por toda a cadeia de chamadas assíncronas de uma requisição. É thread-safe e não requer NestJS request scope, evitando contaminação de escopo nos providers.

#### Métodos

| Método             | Parâmetros                                  | Retorno               | Descrição                                                                         |
| :----------------- | :------------------------------------------ | :-------------------- | :-------------------------------------------------------------------------------- |
| `run()`            | `requesterId: string, fn: () => Promise<T>` | `Promise<T>`          | Executa `fn` dentro de um contexto onde `requesterId` está disponível             |
| `getRequesterId()` | —                                           | `string \| undefined` | Retorna o `requesterId` do contexto atual; `undefined` se chamado fora de `run()` |

## Trechos de Código

### `TrilhaProxyRepository` — proxy de autorização para escrita

> [`backend/src/modules/trilhas/infrastructure/persistence/TrilhaProxyRepository.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/backend/src/modules/trilhas/infrastructure/persistence/TrilhaProxyRepository.ts)

```typescript
export class TrilhaProxyRepository implements ITrilhaRepository {
  constructor(
    private readonly wrapped: ITrilhaRepository,
    private readonly context: TrilhaRequestContext,
  ) {}

  async save(trilha: Trilha): Promise<Trilha> {
    const requesterId = this.context.getRequesterId();
    if (!requesterId || requesterId !== trilha.organizadorId) {
      throw new ForbiddenException(
        "Apenas o organizador pode modificar a trilha",
      );
    }
    return this.wrapped.save(trilha);
  }

  // Leituras passam direto ao repositório real
  findById(id: string): Promise<Trilha | null> {
    return this.wrapped.findById(id);
  }
  findAll(): Promise<Trilha[]> {
    return this.wrapped.findAll();
  }
  create(trilha: Trilha): Promise<Trilha> {
    return this.wrapped.create(trilha);
  }
}
```

### `PontosAuthProxy` — proxy de autorização para pontos turísticos

> [`backend/src/modules/pontos-turisticos/interface/proxies/PontosAuthProxy.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/branch-adapter-object_pool-mediator/backend/src/modules/pontos-turisticos/interface/proxies/PontosAuthProxy.ts)

```typescript
@Injectable()
export class PontosAuthProxy implements IPontosTuristicosService {
  // Busca pública passa direto
  async buscarFeed(filtros: Record<string, any>) {
    return this.servico.buscarFeed(filtros);
  }
  // Escrita valida o usuário antes de delegar
  async criar(dados: any, usuarioId: string) {
    const usuario = await this.userRepository.findByEmail(usuarioId);
    if (!usuario) throw new ForbiddenException("Usuário não encontrado.");
    return this.servico.criar(dados, usuarioId);
  }
}
```

## Rotas Relacionadas

| Rota                     | Método | Descrição                             | Como o Proxy Atua                                                                                           |
| :----------------------- | :----- | :------------------------------------ | :---------------------------------------------------------------------------------------------------------- |
| `/trilhas/:id/finalizar` | `POST` | Finalizar trilha (apenas organizador) | Controller define `context.run(userId, ...)` → `save()` do Proxy verifica `userId === trilha.organizadorId` |
| `/trilhas`               | `GET`  | Listar todas as trilhas               | `findAll()` — Proxy delega livremente sem verificação                                                       |
| `/trilhas`               | `POST` | Criar nova trilha                     | `create()` — Proxy delega livremente; autenticação garantida pelo `JwtAuthGuard`                            |
