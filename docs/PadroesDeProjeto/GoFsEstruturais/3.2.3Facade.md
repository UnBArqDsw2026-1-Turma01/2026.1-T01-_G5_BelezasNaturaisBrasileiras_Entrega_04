## Introdução

O **Facade** é um padrão estrutural que fornece uma interface unificada e simplificada para um conjunto de interfaces em um subsistema. É útil quando você deseja simplificar a complexidade de interação com múltiplos componentes.

Este padrão atua como uma fachada, ocultando a complexidade interna de um subsistema e fornecendo uma interface simples para os clientes.

## Quando Aplicar?

- Quando você deseja fornecer uma interface simples para um subsistema complexo
- Quando há muitas dependências entre clientes e classes de implementação
- Quando você deseja estruturar subsistemas em camadas
- Quando deseja desacoplar clientes de componentes de um subsistema
- Quando múltiplas classes relacionadas precisam ser usadas juntas

## Metodologia

Durante o desenvolvimento do backend do projeto Belezas Naturais Brasileiras, identificamos que os controllers acumulavam dependências diretas com vários Use Cases individuais, violando o princípio de que o controller deve apenas receber e delegar requisições HTTP. O `TrilhasController` possuía 7 dependências injetadas (`CriarTrilhaUseCase`, `ListarTrilhasUseCase`, `FinalizarTrilhaUseCase`, `ListarInscricoesUseCase`, `ConfirmationCodeService`, `TrilhaEventEmitter`, `IBadgeRepository`) e o `InscricoesController` acumulava 5 Use Cases diretamente.

A solução foi aplicar o padrão **Facade** para criar duas fachadas — `InscricaoFacade` e `TrilhaFacade` — cada uma agrupando os Use Cases de seu módulo e fornecendo uma interface única e coesa para o controller correspondente.

A implementação seguiu a metodologia **TDD (Test-Driven Development)**, com os arquivos de teste escritos antes da implementação:

1. Escrita dos specs (`InscricaoFacade.spec.ts` e `TrilhaFacade.spec.ts`) cobrindo delegação correta de parâmetros e propagação de exceções — fase **RED**
2. Implementação das Facades para fazer os testes passarem — fase **GREEN**
3. Refatoração dos controllers para injetar a Facade em vez dos Use Cases avulsos — fase **REFACTOR**

## Estrutura e Participantes

| Classe                      | Papel no Padrão | Responsabilidade                                                                                                                                                                                          |
| :-------------------------- | :-------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `InscricaoFacade`           | **Facade**      | Porta de entrada única para todas as operações de inscrição; agrupa `SolicitarInscricaoUseCase`, `AceitarInscricaoUseCase`, `RejeitarInscricaoUseCase`, `FazerCheckinUseCase` e `ListarInscricoesUseCase` |
| `TrilhaFacade`              | **Facade**      | Porta de entrada única para todas as operações de trilha; agrupa `CriarTrilhaUseCase`, `ListarTrilhasUseCase` e `FinalizarTrilhaUseCase`                                                                  |
| `InscricoesController`      | **Client**      | Recebe as requisições HTTP do módulo de inscrições e delega integralmente ao `InscricaoFacade`, sem conhecer os Use Cases internos                                                                        |
| `TrilhasController`         | **Client**      | Recebe as requisições HTTP do módulo de trilhas e delega as operações de domínio ao `TrilhaFacade`; mantém dependências diretas apenas para operações auxiliares (Composite, Singleton, Badges)           |
| `SolicitarInscricaoUseCase` | **Subsystem**   | Valida se a trilha está ativa, verifica duplicata e cria a inscrição                                                                                                                                      |
| `AceitarInscricaoUseCase`   | **Subsystem**   | Verifica se o solicitante é o organizador, gera o código de confirmação via Singleton e persiste a aceitação                                                                                              |
| `RejeitarInscricaoUseCase`  | **Subsystem**   | Verifica se o solicitante é o organizador e persiste a rejeição                                                                                                                                           |
| `FazerCheckinUseCase`       | **Subsystem**   | Verifica organizador, valida o código via Singleton e marca a presença do participante                                                                                                                    |
| `ListarInscricoesUseCase`   | **Subsystem**   | Retorna inscrições filtradas por usuário (`minhas`) ou por trilha (`porTrilha`)                                                                                                                           |
| `CriarTrilhaUseCase`        | **Subsystem**   | Cria e persiste uma nova trilha com status `ATIVA`                                                                                                                                                        |
| `ListarTrilhasUseCase`      | **Subsystem**   | Retorna todas as trilhas ordenadas por data de criação                                                                                                                                                    |
| `FinalizarTrilhaUseCase`    | **Subsystem**   | Inativa a trilha, busca participantes presentes e emite evento de finalização para os Observers                                                                                                           |

## Descrição das Classes

### `InscricaoFacade`

**Caminho no Projeto:** [`backend/src/modules/inscricoes/application/InscricaoFacade.ts`](../../../backend/src/modules/inscricoes/application/InscricaoFacade.ts)

Fachada que centraliza todas as operações do ciclo de vida de uma inscrição. Antes da sua criação, o `InscricoesController` injetava diretamente os 5 Use Cases, expondo ao controller os detalhes de cada operação. Com a Facade, o controller conhece apenas uma única dependência.

#### Métodos

| Método           | Parâmetros                                                   | Retorno                | Descrição                                                                          |
| :--------------- | :----------------------------------------------------------- | :--------------------- | :--------------------------------------------------------------------------------- |
| `solicitar()`    | `usuarioId: string, trilhaId: string`                        | `Promise<Inscricao>`   | Delega ao `SolicitarInscricaoUseCase` a validação da trilha e criação da inscrição |
| `aceitar()`      | `inscricaoId: string, organizadorId: string`                 | `Promise<Inscricao>`   | Delega ao `AceitarInscricaoUseCase` a validação do organizador e geração do código |
| `rejeitar()`     | `inscricaoId: string, organizadorId: string`                 | `Promise<Inscricao>`   | Delega ao `RejeitarInscricaoUseCase` a validação do organizador e rejeição         |
| `fazerCheckin()` | `inscricaoId: string, organizadorId: string, codigo: string` | `Promise<Inscricao>`   | Delega ao `FazerCheckinUseCase` a validação do código e marcação de presença       |
| `minhas()`       | `usuarioId: string`                                          | `Promise<Inscricao[]>` | Delega ao `ListarInscricoesUseCase.minhas()` a busca das inscrições do usuário     |

---

### `TrilhaFacade`

**Caminho no Projeto:** [`backend/src/modules/trilhas/application/TrilhaFacade.ts`](../../../backend/src/modules/trilhas/application/TrilhaFacade.ts)

Fachada que centraliza as operações de domínio de trilha. O `TrilhasController` mantém dependências diretas apenas para funcionalidades auxiliares que não pertencem ao domínio de trilha (Composite de localização, Singleton de código, Badges, status do sistema).

#### Métodos

| Método        | Parâmetros                                       | Retorno             | Descrição                                                                       |
| :------------ | :----------------------------------------------- | :------------------ | :------------------------------------------------------------------------------ |
| `criar()`     | `organizadorId: string, input: CriarTrilhaInput` | `Promise<Trilha>`   | Delega ao `CriarTrilhaUseCase` a criação e persistência da trilha               |
| `listar()`    | —                                                | `Promise<Trilha[]>` | Delega ao `ListarTrilhasUseCase` a listagem de todas as trilhas                 |
| `finalizar()` | `trilhaId: string, organizadorId: string`        | `Promise<void>`     | Delega ao `FinalizarTrilhaUseCase` a inativação e notificação dos participantes |

## Trechos de Código

### `InscricaoFacade` — porta de entrada única para os 5 use cases de inscrição

> [`backend/src/modules/inscricoes/application/InscricaoFacade.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/backend/src/modules/inscricoes/application/InscricaoFacade.ts)

```typescript
@Injectable()
export class InscricaoFacade {
  constructor(
    private readonly solicitarUC: SolicitarInscricaoUseCase,
    private readonly aceitarUC: AceitarInscricaoUseCase,
    private readonly rejeitarUC: RejeitarInscricaoUseCase,
    private readonly fazerCheckinUC: FazerCheckinUseCase,
    private readonly listarUC: ListarInscricoesUseCase,
  ) {}

  solicitar(usuarioId: string, trilhaId: string): Promise<Inscricao> {
    return this.solicitarUC.execute(usuarioId, trilhaId);
  }
  aceitar(inscricaoId: string, organizadorId: string): Promise<Inscricao> {
    return this.aceitarUC.execute(inscricaoId, organizadorId);
  }
  fazerCheckin(
    inscricaoId: string,
    organizadorId: string,
    codigo: string,
  ): Promise<Inscricao> {
    return this.fazerCheckinUC.execute(inscricaoId, organizadorId, codigo);
  }
  minhas(usuarioId: string): Promise<Inscricao[]> {
    return this.listarUC.minhas(usuarioId);
  }
}
```

### `TrilhaFacade` — porta de entrada única para os 3 use cases de trilha

> [`backend/src/modules/trilhas/application/TrilhaFacade.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/backend/src/modules/trilhas/application/TrilhaFacade.ts)

```typescript
@Injectable()
export class TrilhaFacade {
  criar(organizadorId: string, input: CriarTrilhaInput): Promise<Trilha> {
    return this.criarUC.execute(organizadorId, input);
  }
  listar(input?: ListarTrilhasInput): Promise<Trilha[]> {
    return this.listarUC.execute(input);
  }
  finalizar(trilhaId: string, organizadorId: string): Promise<void> {
    return this.finalizarUC.execute(trilhaId, organizadorId);
  }
}
```

## Rotas Relacionadas

| Rota                           | Método | Descrição                                            | Como Testar                                                                                                |
| :----------------------------- | :----- | :--------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- |
| `/inscricoes/trilha/:trilhaId` | `POST` | Solicitar vaga em trilha                             | JWT de usuário autenticado + `trilhaId` de trilha ativa no path                                            |
| `/inscricoes/minhas`           | `GET`  | Listar inscrições do usuário autenticado             | JWT de usuário autenticado                                                                                 |
| `/inscricoes/:id/aceitar`      | `POST` | Aceitar inscrição (apenas organizador da trilha)     | JWT do organizador + `id` da inscrição pendente no path                                                    |
| `/inscricoes/:id/rejeitar`     | `POST` | Rejeitar inscrição (apenas organizador da trilha)    | JWT do organizador + `id` da inscrição pendente no path                                                    |
| `/inscricoes/:id/checkin`      | `POST` | Confirmar presença no ponto de encontro              | JWT do organizador + `id` da inscrição aceita + `{ "codigo": "XXXXXXXX" }` no body                         |
| `/trilhas`                     | `GET`  | Listar todas as trilhas cadastradas                  | Sem autenticação                                                                                           |
| `/trilhas`                     | `POST` | Criar nova trilha                                    | JWT de usuário autenticado + body com `titulo`, `descricao`, `pontoEncontro`, `dataInicio`, `vagasMaximas` |
| `/trilhas/:id/finalizar`       | `POST` | Finalizar trilha e notificar participantes presentes | JWT do organizador da trilha + `id` da trilha no path                                                      |
