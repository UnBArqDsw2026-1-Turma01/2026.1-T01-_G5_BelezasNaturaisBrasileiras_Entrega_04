## Introdução

Como iniciativa extra, a equipe construiu uma suíte abrangente de testes para validar os padrões de projeto implementados e os fluxos de negócio do sistema. Os testes estão organizados em três camadas:

1. **Testes unitários/integração com Jest** — validam padrões GoF de forma isolada
2. **Script de integração end-to-end** — exercita o fluxo completo do sistema (40 passos, cobrindo todos os 18 padrões implementados)
3. **Guia de testes manuais** — roteiro de `curl` para execução manual no terminal

## Script de Integração End-to-End

O script [`run_integration_tests.ps1`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/run_integration_tests.ps1) executa **40 passos** que cobrem todos os 18 padrões GoF implementados em um único fluxo automatizado, do cadastro ao check-in, passando por todos os módulos do sistema.

### Fluxo dos 40 Passos

| Passos  | Seção                                                       | Padrões Exercitados                                                           |
| :------ | :---------------------------------------------------------- | :---------------------------------------------------------------------------- |
| 1–4     | Contas: signup (organizador, participante, admin) + login   | Chain of Responsibility, Factory Method, Abstract Factory, Builder, Prototype |
| 5–6     | Trilhas: criar e listar (com filtro e ordenação)            | Builder, Iterator, Strategy                                                   |
| 7       | Composite: criar hierarquia geográfica de ponto de encontro | Composite                                                                     |
| 8a–8c   | Singleton: gerar código, validar e checar status            | Singleton                                                                     |
| 9       | Participante solicita inscrição                             | State (PENDENTE), Facade                                                      |
| 10      | Organizador lista inscrições                                | Iterator, Facade                                                              |
| 11      | Organizador aceita inscrição                                | State (ACEITA), Facade                                                        |
| 12      | Organizador faz check-in                                    | State (PRESENTE), Facade                                                      |
| 13      | Editar trilha (salva snapshot antes)                        | Command, Memento                                                              |
| 14      | Restaurar estado anterior da trilha                         | Memento                                                                       |
| 15      | Finalizar trilha                                            | State (INATIVA), Observer                                                     |
| 16      | Verificar badges recebidos                                  | Observer                                                                      |
| 17      | Listar minhas inscrições                                    | Facade                                                                        |
| 18      | Criar ponto turístico                                       | Proxy (auth)                                                                  |
| 19a–19c | Listar pontos (cache miss, cache hit, com filtro)           | Proxy (cache)                                                                 |
| 20      | Editar ponto turístico                                      | Proxy (auth)                                                                  |
| 21      | Status do pool de conexões de chat                          | Object Pool                                                                   |
| 22–24   | Chat: iniciar sessão, enviar mensagem, encerrar             | Object Pool                                                                   |
| 25–28   | Adapters: info, geocode, route, SMS, WhatsApp, auth         | Adapter                                                                       |
| 29      | Finalizar ponto turístico                                   | Mediator                                                                      |
| 30      | Deletar ponto turístico                                     | Proxy (auth)                                                                  |
| 31a–31b | Admin promove usuário de role                               | Prototype                                                                     |

### Como Executar (PowerShell — Windows)

> O backend deve estar rodando em `http://localhost:3000`

```powershell
# Na raiz do repositório
.\run_integration_tests.ps1
```

### Como Executar (bash — macOS/Linux)

Use o guia de testes manuais disponível em [`CURLS_MANUAL.md`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/CURLS_MANUAL.md), que replica os mesmos 40+ passos como comandos `curl` sequenciais.

```bash
# Exemplo: criar conta do organizador
curl -X POST http://localhost:3000/accounts/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "org@teste.com",
    "password": "Senha@123",
    "nome": "Organizador",
    "role": "ORGANIZER",
    "aceitouTermos": true
  }'
```
