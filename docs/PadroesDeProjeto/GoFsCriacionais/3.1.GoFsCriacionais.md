# 3.1 Padrões de Projeto Criacionais (GoFs Criacionais)

## Introdução

Os Padrões de Projeto Criacionais (Creational Design Patterns) são um conjunto de soluções reutilizáveis que lidam com os mecanismos de criação de objetos, abstraindo a lógica de instanciação para tornar um sistema independente de como seus objetos são compostos e representados. 

A importância dos padrões criacionais reside em que eles permitem que o código cliente não dependa das classes concretas dos objetos que precisam ser criados. Em vez disso, o cliente trabalha com abstrações (interfaces ou classes abstratas), o que reduz o acoplamento, aumenta a flexibilidade e facilita a extensão do sistema sem modificar o código existente.

## Padrões Criacionais

| Padrão | Descrição | Onde Aplicar no Código | Benefícios |
| :--- | :--- | :--- | :--- |
| **Factory Method** | Define uma interface para criar um objeto, permitindo que subclasses decidam qual classe instanciar. | | |
| **Singleton** | Garante que uma classe tenha apenas uma instância e fornece um ponto de acesso global a ela. | | |
| **Prototype** | Cria novos objetos clonando um objeto protótipo, em vez de criá-los do zero. | | |
| **Abstract Factory** | Fornece uma interface para criar famílias de objetos relacionados sem especificar suas classes concretas. | | |
| **Builder** | Constrói um objeto complexo passo a passo, separando a construção de sua representação. | | |
