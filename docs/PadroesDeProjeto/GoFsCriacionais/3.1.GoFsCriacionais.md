# 3.1 Padrões de Projeto Criacionais (GoFs Criacionais)

## Introdução

Os Padrões de Projeto Criacionais (Creational Design Patterns) fornecem vários mecanismos de criação de objetos, que aumentam a flexibilidade e a reutilização de código já existente.

A ideia fundamental por trás desses padrões é abstrair o processo de instanciação. Eles ajudam a tornar um sistema independente de como seus objetos são criados, compostos e representados. Isso é crucial no sistema Belezas Naturais Brasileiras para garantir que o código cliente não precise conhecer as classes concretas dos objetos que está criando, facilitando a manutenção e permitindo que o sistema cresça sem a necessidade de refatorações constantes em múltiplos pontos.

## Padrões Criacionais

| Padrão               | Descrição                                                                                                             |     Status      | Onde Aplicar no Código                                                                                                                                                                                                                                                                                                                                                                    | Benefícios                                                                                                                                                               |
| :------------------- | :-------------------------------------------------------------------------------------------------------------------- | :-------------: | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Factory Method**   | Fornece uma interface para criar objetos em uma superclasse, mas permite que as subclasses alterem o tipo de objetos que serão criados. | ✅ Implementado | Criação de usuários por role (`CommonUserFactory`, `OrganizerUserFactory`, `AdminUserFactory`) e distribuição de diferentes tipos de `Badge` ao finalizar uma trilha.                                                                                                                                                                                                                    | Desacopla a lógica de criação da lógica de negócio; facilita a adição de novos tipos de produtos sem quebrar o código existente.                                         |
| **Singleton**        | Garante que uma classe tenha apenas uma instância e fornece um ponto de acesso global para essa instância.           | ✅ Implementado | `ConfirmationCodeService` — garante uma única instância do serviço que gera, valida e revoga o código único de presença.                                                                                                                                                                                                                                                                  | Economia de recursos e garantia de que o estado global (como o controle de códigos de acesso) permaneça consistente em toda a aplicação.                                 |
| **Prototype**        | Permite copiar objetos existentes sem fazer seu código depender de suas classes.                                     | ✅ Implementado | `User.clone()` e duplicação de trilhas existentes para criar novas edições com o mesmo ponto de encontro e estrutura base.                                                                                                                                                                                                                                                                | Permite clonar objetos complexos sem acoplar o código às suas classes concretas; simplifica a criação de variações de um mesmo objeto base.                              |
| **Abstract Factory** | Permite produzir famílias de objetos relacionados sem especificar suas classes concretas.                            | ✅ Implementado | `UserFactoryRegistry` — registro central que mapeia cada `UserRole` para sua `IUserFactory` correspondente, garantindo a criação consistente de famílias de usuários.                                                                                                                                                                                                                    | Garante que os produtos criados sejam compatíveis entre si; promove o desacoplamento entre o código cliente e os detalhes de implementação das fábricas.                 |
| **Builder**          | Permite a construção de objetos complexos passo a passo. O padrão permite produzir diferentes tipos e representações de um objeto usando o mesmo código de construção. | ✅ Implementado | `UserBuilder` e `TrilhaBuilder` para montagem de entidades com muitos campos opcionais (data, vagas, ponto de encontro, dificuldade).                                                                                                                                                                                                                                                    | Melhora a legibilidade do código de criação; permite a construção incremental de objetos e a validação de dados antes da instanciação final.                             |
| **Object Pool**      | Gerencia um conjunto de objetos que são mantidos prontos para uso em vez de serem criados e destruídos sob demanda.  | ✅ Implementado | `ChatObjectPoolService` gerencia conexões de chat reutilizáveis para otimizar o desempenho do sistema de mensagens em tempo real.                                                                                                                                                                                                                                                         | Reduz drasticamente o custo de desempenho associado à criação e destruição frequente de objetos pesados (como conexões de rede ou banco de dados).                       |

> **Legenda de Status:** ✅ Implementado — já presente no código &nbsp;|&nbsp; ⚠️ Expansão de escopo — requer nova funcionalidade &nbsp;|&nbsp; ❌ Planejado — previsto para implementação no escopo atual
