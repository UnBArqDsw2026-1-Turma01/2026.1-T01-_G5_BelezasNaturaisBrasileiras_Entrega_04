## Introdução

O **Prototype** é um padrão criacional que permite criar novos objetos clonando um objeto protótipo, em vez de criá-los do zero. É particularmente útil quando a criação de um objeto é cara, complexa ou quando você deseja evitar a dependência de classes concretas.

Este padrão define uma interface para clonar um objeto, permitindo que diferentes tipos de objetos sejam duplicados sem conhecer seus detalhes internos de construção.

## Quando Aplicar?

- Quando criar uma instância é custoso e você pode economizar clonando um objeto já existente
- Quando há muitas subclasses e você quer evitar uma hierarquia de classes de criação
- Quando a configuração inicial de um objeto é complexa
- Quando você precisa de cópias independentes de objetos com estado pré-configurado
- Quando deseja implementar um sistema de undo/redo através de snapshots de estado

## Metodologia

O padrão Prototype foi aplicado à entidade `User` para resolver o problema de **promoção de role**. Quando um administrador promove um usuário (ex.: de `COMMON_USER` para `ORGANIZER`), o objeto `User` existente não deve ser modificado diretamente — isso violaria a imutabilidade do domínio. Criar um novo `User` do zero duplicaria toda a lógica de construção e dependeria de conhecer todos os campos do objeto original.

Com o Prototype, `user.clone({ role: newRole, updatedAt: new Date() })` cria uma cópia do objeto substituindo apenas os campos informados. O `PromoteUserUseCase` busca o usuário no banco, clona com a nova role, e persiste o clone — sem expor os detalhes internos do `User` nem duplicar lógica de construção.

A interface `IPrototype<T>` define o contrato genérico de clonagem, mantendo o domínio desacoplado de implementações concretas.

## Descrição das Classes

**`IPrototype<T>`** (`domain/interfaces/IPrototype.ts`)

Interface genérica do padrão. Define o método `clone(overrides?: Partial<T>): T`, onde `overrides` permite substituir seletivamente qualquer campo da cópia sem precisar informar todos os valores do objeto original.

**`User`** (`domain/entities/User.ts`)

Prototype concreto. O método `clone()` usa operador de coalescência nula (`??`) para cada campo: se o override foi fornecido, usa o novo valor; caso contrário, mantém o valor do objeto original. Isso garante que a cópia seja independente do original (nenhuma referência compartilhada).

```typescript
export interface IPrototype<T> {
  clone(overrides?: Partial<T>): T;
}
```

### `User.clone()` — cópia profunda com sobrescrita parcial

> [`backend/src/modules/accounts/domain/entities/User.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/backend/src/modules/accounts/domain/entities/User.ts)

```typescript
export class User implements IPrototype<User> {
  // ...
  clone(overrides?: Partial<User>): User {
    return new User(
      overrides?.id ?? this.id,
      overrides?.email ?? this.email,
      overrides?.nome ?? this.nome,
      overrides?.role ?? this.role,
      overrides?.fotoPerfil !== undefined
        ? overrides.fotoPerfil
        : this.fotoPerfil,
      overrides?.createdAt ?? this.createdAt,
      overrides?.updatedAt ?? this.updatedAt,
    );
  }
}
```
