## Introdução

O **Factory Method** é um padrão de projeto criacional que fornece uma interface para criar objetos em uma superclasse, mas permite que as subclasses alterem o tipo de objetos que serão criados.

O problema central que este padrão resolve é a necessidade de criar objetos sem especificar as classes exatas dos objetos que serão produzidos. Ao mover a lógica de instanciação para métodos especializados (os métodos de fábrica), o sistema torna-se mais flexível, permitindo que o código cliente dependa de abstrações em vez de implementações concretas, o que facilita a extensão do sistema com novos tipos de produtos.

## Quando Aplicar?

- Quando você não sabe antecipadamente os tipos e dependências exatas dos objetos com os quais seu código deve funcionar.
- Quando você deseja fornecer aos usuários da sua biblioteca ou framework uma maneira de estender seus componentes internos.
- Quando você deseja economizar recursos do sistema reutilizando objetos existentes em vez de reconstruí-los todas as vezes.
- Para desacoplar a lógica de criação da lógica de negócio principal.

## Metodologia

No sistema Belezas Naturais Brasileiras, o padrão Factory Method foi adotado para a **gestão de criação de usuários baseada em perfis (roles)**. O sistema suporta diferentes categorias de usuários (`COMMON_USER`, `ORGANIZER`, `ADMIN`), cada um exigindo configurações de permissões e estados iniciais distintos.

A implementação utiliza uma interface `IUserFactory` que declara o método `create`. Cada factory concreta (ex: `CommonUserFactory`) implementa esse método para produzir uma instância de `User` com as características específicas daquela role. 

Esta abordagem permite que os Use Cases da aplicação permaneçam agnósticos quanto ao processo de construção dos usuários. Se no futuro for necessário adicionar um novo tipo de perfil (ex: `MODERATOR`), basta criar uma nova classe factory sem precisar alterar a lógica de cadastro ou promoção existente, respeitando o Princípio Aberto/Fechado.

## Estrutura e Participantes

| Classe                 | Papel no Padrão        | Responsabilidade                                  |
| :--------------------- | :--------------------- | :------------------------------------------------ |
| `IUserFactory`         | Creator (Interface)    | Declara o método de fábrica que retorna o objeto do tipo `User`. |
| `CommonUserFactory`    | Concrete Creator       | Sobrescreve o método de fábrica para criar usuários comuns.        |
| `OrganizerUserFactory` | Concrete Creator       | Sobrescreve o método de fábrica para criar organizadores.         |
| `AdminUserFactory`     | Concrete Creator       | Sobrescreve o método de fábrica para criar administradores.        |
| `User`                 | Product (Interface)    | Define a interface dos objetos que o método de fábrica cria.      |

## Descrição das Classes

**`IUserFactory`** (`domain/interfaces/IUserFactory.ts`)

Interface do creator. Define o método `create(id, email, nome): User` que todas as factories devem implementar. Desacopla o código cliente das implementações concretas.

**`CommonUserFactory`** (`infrastructure/factories/CommonUserFactory.ts`)

Factory para usuários comuns. Usa `UserBuilder` configurando `role = UserRole.COMMON_USER`. É a factory padrão usada no fluxo de cadastro (`CreateAccountUseCase`).

**`OrganizerUserFactory`** (`infrastructure/factories/OrganizerUserFactory.ts`)

Factory para organizadores. Usa `UserBuilder` configurando `role = UserRole.ORGANIZER`. Usada quando um usuário é promovido para poder criar trilhas.

**`AdminUserFactory`** (`infrastructure/factories/AdminUserFactory.ts`)

Factory para administradores. Usa `UserBuilder` configurando `role = UserRole.ADMIN`. Usada em promoções administrativas.

}

### Fábrica Concreta — `CommonUserFactory`

> [`backend/src/modules/accounts/infrastructure/factories/CommonUserFactory.ts`](https://github.com/UnBArqDsw2026-1-Turma01/2026.1-T01-_G5_BelezasNaturaisBrasileiras_Entrega_01/blob/main/backend/src/modules/accounts/infrastructure/factories/CommonUserFactory.ts)

```typescript
@Injectable()
export class CommonUserFactory implements IUserFactory {
  create(id: string, email: string, nome: string): User {
    return new UserBuilder()
      .withId(id)
      .withEmail(email)
      .withNome(nome)
      .withRole(UserRole.COMMON_USER)
      .build();
  }
}
```
